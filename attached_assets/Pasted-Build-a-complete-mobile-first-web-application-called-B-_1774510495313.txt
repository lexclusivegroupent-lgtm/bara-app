Build a complete mobile-first web application called Bära — a two-sided gig marketplace for on-demand furniture transport and junk pickup in Sweden. Build this as a Progressive Web App (PWA) using React for the frontend, Node.js with Express for the backend, and PostgreSQL for the database. The app should work perfectly in a mobile browser and feel like a native app.

TECH STACK
	∙	Frontend: React with Tailwind CSS
	∙	Backend: Node.js with Express
	∙	Database: PostgreSQL
	∙	Authentication: JWT tokens + Google OAuth
	∙	Maps: Google Maps JavaScript API + Distance Matrix API
	∙	Payments: Stripe (SEK currency)
	∙	Real-time: Socket.io for live job updates
	∙	Push notifications: Web Push API
	∙	Hosting: Replit

BRAND IDENTITY
	∙	App name: Bära
	∙	Tagline: “Carry anything, anywhere” / “Bär vad som helst, vart som helst”
	∙	Primary color: Navy #1B2A4A
	∙	Accent color: Gold/Orange #C9A84C
	∙	Font: Inter
	∙	Logo: Delivery truck icon in gold on navy background

USER TYPES
Two user types — Customer and Driver. Users select their role during signup. A user can only be one role.

DATABASE SCHEMA
Users table:
	∙	id, email, password_hash, full_name, role (customer/driver), city, profile_photo, is_available, rating, total_jobs, vehicle_description, address, created_at
Jobs table:
	∙	id, customer_id, driver_id, job_type (furniture_transport/junk_pickup), status (pending/accepted/in_progress/completed/cancelled), pickup_address, dropoff_address, home_address, item_description, preferred_time, distance_km, price_total, driver_payout, platform_fee, rating, payment_status (unpaid/paid), city, photos_pickup, photos_dropoff, created_at, accepted_at, completed_at
Ratings table:
	∙	id, job_id, rater_id, rated_id, score, comment, created_at

PRICING LOGIC
	∙	Furniture Transport: base price 299 SEK + 15 SEK per km
	∙	Junk and Trash Pickup: base price 199 SEK + 10 SEK per km
	∙	Minimum price for any job: 349 SEK regardless of distance
	∙	Formula: price_total = maximum of (349, base_price + (distance_km × rate_per_km))
	∙	Driver receives 75% of price_total
	∙	Platform keeps 25% of price_total
	∙	Stripe processing fee approximately 1.4% + 1.80 SEK per transaction

WORKFLOWS AND BUSINESS LOGIC
Job Posting Workflow
	1.	Customer fills out job form and taps Post Job
	2.	App calls Google Distance Matrix API with pickup and dropoff addresses
	3.	Distance in meters is returned, divide by 1000 to get km, save to distance_km field
	4.	Calculate price_total using formula: maximum of (349, base_price + distance_km × rate)
	5.	Calculate driver_payout = price_total × 0.75
	6.	Calculate platform_fee = price_total × 0.25
	7.	Save job to database with status Pending and current user’s city
	8.	Emit Socket.io event to notify nearby drivers of new job
	9.	Navigate customer to My Jobs screen
Job Acceptance Workflow
	1.	Driver taps Accept on a job card
	2.	Update job status to Accepted
	3.	Set driver_id to current user id
	4.	Set accepted_at timestamp
	5.	Emit Socket.io event to notify customer their job was accepted
	6.	Navigate driver to Driver Active Job screen with job data
Job Completion Workflow
	1.	Driver taps Complete Job
	2.	Update job status to Completed
	3.	Set completed_at timestamp
	4.	Process Stripe payment — charge customer price_total in SEK
	5.	Update payment_status to paid
	6.	Navigate to Rate User screen
Rating Workflow
	1.	User taps star rating 1 through 5
	2.	Selected rating is stored in component state
	3.	User taps Submit Rating
	4.	Save rating score and comment to ratings table
	5.	Update the rated user’s average rating in users table
	6.	Update job record with rating value
	7.	Navigate to CustomerHome if customer or DriverMap if driver

REAL-TIME FEATURES
Use Socket.io for:
	∙	New job posted → emit to all online drivers in same city
	∙	Job accepted → emit to customer who posted the job
	∙	Job completed → emit to both customer and driver
